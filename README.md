# DevSecOps учебные лабораторные

Этот репозиторий содержит учебные материалы по DevSecOps и AppSec: разбор поверхности атаки, анализ уязвимостей, proof-of-concept payload'ы для специально подготовленных стендов и рекомендации по исправлению.

Материалы предназначены для обучения и защиты лабораторных работ. Они применяются только к открытым учебным проектам и локальным/выданным стендам, где анализ разрешён.

## Лабораторная работа №1

Тема: развёртывание уязвимого веб-приложения и анализ поверхности атаки.

Выбранный проект: [vulnerable-apps/vuln_django_play](https://github.com/vulnerable-apps/vuln_django_play).

Цель лабораторной работы №1 — не эксплуатация, а inventory:

- описать состав системы;
- зафиксировать сетевые точки доступа;
- найти программные точки входа;
- выделить активы и ценные данные;
- описать роли, аутентификацию и секреты без раскрытия значений;
- показать границы доверия.

Материалы:

## Лабораторная работа №2

Тема: анализ и эксплуатация типовых web-уязвимостей на учебных стендах.

В работе разбираются 8 классов уязвимостей:

- CSRF;
- unrestricted file upload до RCE;
- IDOR через клиентскую роль;
- broken access control из-за ошибки логического условия;
- path traversal;
- blind SQL injection;
- SSRF через обход blacklist;
- reflected XSS через type confusion.

## Этические рамки

Все примеры предназначены для:

- локальных учебных стендов;
- специально уязвимых приложений;
- систем, где владелец явно разрешил тестирование.

Не использовать эти payload'ы против чужих систем без письменного разрешения. Цель материалов — понять, как уязвимости возникают, как их доказать в контролируемой среде и как исправлять класс проблемы.

## Подход к оформлению

Для каждой уязвимости фиксируются:

1. адрес стенда и класс уязвимости;
2. точки входа и наблюдаемый стек;
3. фрагмент уязвимой логики;
4. payload или теоретический PoC;
5. механика обхода защиты;
6. корректное исправление.

Секреты и пароли в отчётах не публикуются. Если в конфигурациях встречаются секреты, в отчёт заносится только имя переменной, место хранения и риск.
